$70:0F00 - $70:0F5F 96 bytes total Table of all current regular sprites' states. Each sprite entry is 4 bytes - only 2 are used for state. Therefore there may be something else in each entry, unknown.
This means there are only 24 regular sprites possible at a time in the game.


There are 9 sprite states in the sprite_state_routines table, and 0 is used for "none", so 10 total. The states act as indices into the table as well, so they go by 2's.
Known sprite states (in hex):
$0000 - nonexistent (used for skipping processing)
$0002 - newly spawned
$0004 - unknown
$0006 - unknown
$0008 - unknown
$000A - unknown
$000C - unknown
$000E - unknown
$0010 - active / alive
$0012 - unknown
$70:1360 - $70:13BF 96 bytes total Table of all current regular sprites' ID's. Each sprite entry is 4 bytes - only 2 are used for ID. Therefore there may be something else in each entry, unknown.
$7E:0C04 - $7E:0C0B 8 bytes total table of all current hidden sprites' ID's: 4 2-byte entries representing hidden sprite ID's: 0 is used as "none", anything else is assumed to be an ID. Hidden sprite ID refers to: global sprite ID - 0x1B9 (1-based, as 0 means "none")
$7E:0C1C 2 bytes stores the current autoscroll hidden sprite ID, 0 means no autoscroll active
$7E:0C1E 2 bytes seems to be the same as $0C1C
$7E:0C20 2 bytes boolean - autoscroller takes effect on camera Y - not just X (1-E and 6-8 keep this off)
$7E:0C22 4 bytes 32-bit current autoscroll camera x value
$7E:0C26 4 bytes 32-bit current autoscroll camera y value
$7E:0C2A 2 bytes current autoscroll x velocity
$7E:0C2C 2 bytes current autoscroll y velocity
$7E:0C2E 2 bytes current autoscroll checkpoint (index into autoscroller_values table)
$7E:0C30 2 bytes next autoscroll checkpoint x value
$7E:0C32 2 bytes next autoscroll checkpoint y value
$7E:0C34 2 bytes next autoscroll checkpoint speed value
$7E:0C36 2 bytes autoscroll: next checkpoint x - current camera x (how much is left to go)
$7E:0C38 2 bytes autoscroll: next checkpoint y - current camera y (how much is left to go)
$7E:0C3A 2 bytes boolean - currently active gusty generation
$7E:0C3C 2 bytes boolean - (at least 1) currently active lakitu
$7E:0C3E 2 bytes boolean - currently active fuzzy generation
$7E:0C44 2 bytes previous camera x value for fuzzy generator

$7E:0C4A 2 bytes current generated bat count
